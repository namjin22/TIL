- `프로토콜이란?`

![프로토콜 이미지](https://velog.velcdn.com/images/blessoms2017/post/02658df6-7dfc-4948-a460-18dbfd59901e/image.png)

  - 컴퓨터나 네트워크 장치들이 서로 통신할 때 기본적으로 지키기로 약속한 규칙의 집합이다.

        초기 컴퓨터 네트워킹 시대에 다양한 컴퓨터들이 서로 다른 방식으로 통신을 했었는데, 이로 인해 통신에 다양한 문제가 발생했었다.

        이를 해결하기 위해 표준화된 통신 규칙, 즉 TCP/IP, HTTP, FTP와 같은 표준 프로토콜들이 개발되었다.

        프로토콜은 데이터가 어떤 구조로 전송될지, 데이터가 어디로 전송 되는지,
        통신의 연결 설정 및 해제, 데이터 전송과 흐름 제어, 오류 감지 등의 역할을 수행한다.

- `ip 주소`

  - 컴퓨터 네트워크에서 장치들이 서로를 인식하고 통신을 하기 위해서 사용하는 각 장치마다 할당된 고유한 숫자 레이블로 만들어진 주소이다.

        IPv4 주소는 마침표로 구분된 4개의 숫자로 구성되어 있다. 각 숫자들은 32비트로 0~255의 범위를 갖는다.
  
        (0.0.0.0 부터 255.255.255.255 까지 나타낼 수 있다.) 약 43억에 가까운 수의 IP가 존재한다.

        요즘엔 IPv4 주소의 수가 부족해져서 IPv6이 등장했다.
        
        IPv6는 주소 길이를 128비트로 늘려 사용가능한 주소의 갯수가 2의 128제곱개 정도 된다.

        즉 주소가 약 43억x43억x43억x43억개이다.
        
        IPv6 주소는 보통 두 자리 16진수 여덟 개를 쓰고 각각을 : 기호로 구분한다. (ex. 2001:0db8:85a3:0000:0000:8a2e:0370:7334)

- `mac 주소?`

  - 네트워크 장치에 통신을 위해 부여되는 고유한 주소이다.

        LAN 통신, ARP 프로토콜, 디바이스 식별, 패킷 전달 등에 사용된다.

        MAC 주소는 48비트(6바이트)이며 16진수로 표기한다.(ex. 00:1A:2B:3C:4D:5E)
        
        앞 24비트는 제조사 식별 번호(OUI), 뒤 24비트는 장치 고유 번호이다.

        OSI 7계층 중 데이터 링크 계층인 2계층에서 사용한다.

        로컬 네트워크(LAN) 범위 내에서만 유효하게 사용된다.

- `스위칭 개념`

![스위칭 이미지](https://blog.kakaocdn.net/dn/ZZd0Z/btrkelqrQAI/E6h8e09hLz2w7WeKI2K9MK/img.png)

  - 스위칭은 네트워크에서 데이터가 목적지까지 효율적으로 전달되도록 경로를 선택하고 전송하는 방식이다.

        크게 회선 교환(Circuit Switching), 패킷 교환(Packet Switching), 메시지 교환(Message Switching) 세 가지 방식이 있다.

        ▸ 회선 교환 (Circuit Switching)

        데이터를 주고받기 전에 전용 통신 경로를 설정한다. 경로가 한번 설정되면 통신이 끝날 때까지 고정된다.
        우리가 평소에 하는 전화 통신과 비슷하다고 생각하면 될 것 같다.

        지연이 적어서 실시간 전송에 적합하지만
        통신 중 데이터가 없더라도 전용 회선을 계속 점유하기 때문에 자원을 효율적으로 활용하지 못한다.

        ▸ 패킷 교환 (Packet Switching)

        데이터를 작은 패킷 단위로 나눠서 독립적으로 전송하며 전송된 각 패킷은 다른 경로를 거쳐 도착 후 재조립된다.

        자원을 효율적으로 사용하고 확장성이 좋지만 약간의 지연이나 순서 꼬임 발생 가능성이 있다.
        패킷 교환은 TCP/IP 같은 인터넷에서 가장 널리 쓰이는 방식이다.

        ▸ 메시지 교환 (Message Switching)

        전체 메시지를 받아서 저장 후 다음 노드로 전달하는 방식이다.
        
        회선 미사용 시 자원 낭비가 거의 없다. 하지만 전체 메시지를 저장해야 하므로 지연이 크다.
        메시지 교환 특성상 지연이 크기 때문에 실시간 통신엔 부적합하다.

- `ARP 프로토콜`

![ARP 프로토콜 이미지](https://velog.velcdn.com/images/hyungjin1124/post/181cb975-5985-49f3-818b-999447319c60/image.png)

  - ARP (Address Resolution Protocol) 프로토콜은 네트워크에서 IP 주소를 MAC 주소로 변환해주는 프로토콜이다.

        IP 주소만 알고 있을 때 실제 물리적인 목적지 장치의 MAC 주소를 알아야 데이터 링크 계층 (2계층)에서 통신할 수 있는데,
        이때 ARP 프로토콜이 사용된다.

        같은 네트워크(LAN) 내에서 통신 시 사용되며 같은 주소에 대해 반복해서 요청하지 않도록
        ARP 프로토콜 결과는 일정 시간 동안 메모리에 저장 된다. (캐시)

- `서브넷`

    ![subnet img](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcTIvA5gt7Er_qRo2cKW7W__K6vXPgtEsDnDmA&s)

  - 서브넷(Subnet)은 하나의 IP 네트워크를 여러 개의 작은 네트워크(서브 네트워크)로 나누는 기술이다. 큰 네트워크를 나눠서 주소 공간을 효율적으로 사용하고, 네트워크 관리 및 보안성을 향상시키는 데 쓰인다.

        IP 주소 낭비를 줄이고 네트워크 트래픽을 분산하고 관리하며 보안 구역을 나누는 등 다양한 장점과 필요에 의해 사용된다.

        서브넷은 각 장치의 IP 주소를 기준으로 네트워크 부분과 호스트 부분을 나누는 방식을 사용한다.
        
        이 구분을 위해 사용되는 것이 서브넷 마스크(Subnet Mask)이며,
        서브넷 마스크는 IP 주소 중 어느 부분이 네트워크인지 나타낸다.
        
        예를 들어, 서브넷 마스크가 255.255.255.0이면 `/24`로 표현하고, 
        이는 IP 주소 중 앞의 24비트가 네트워크 주소, 뒤의 8비트가 호스트 주소라는 뜻이다.

- `프라이빗 IP, 퍼블릭 IP`
  
![프라이빗 IP, 퍼블릭 IP 이미지](https://img.etnews.com/news/article/2023/03/17/cms_temp_article_17082343816517.png)

  - `프라이빗 IP`

    - 프라이빗 IP는 내부(사설) 네트워크에서 사용되는 IP 주소로 인터넷과 직접 연결되지는 않는다.

          프라이빗 IP는 라우터나 NAT(Network Address Translation)를 통해
          퍼블릭 IP로 변환해야지 인터넷에 접속할 수 있다.
    
          주로 내부 네트워크에서 내부 장치 간의 통신, 보안상 이점 등의 용도로 사용된다. 

  - `퍼블릭 IP`

    - 퍼블릭 IP는 인터넷에서 고유하게 식별되는 IP 주소로, 전 세계에서 유일하게 사용된다.

          인터넷에 연결된 장치에 모두 할당되며 퍼블릭 IP를 이용하여 인터넷 상의 모든 장치와 통신을 할 수 있다.

          주로 웹 서버, 이메일 서버 등과 같은 외부에서 접근(통신)이 필요한 것들을 만들 때 사용한다.

##### (IPv4) IP 주소가 부족한 문제를 해결하며

##### 네트워크 보안이 강해지고 효율적인 네트워크 관리를

##### 할 수 있는 특성들을 가지기 위해 프라이빗 IP, 퍼블릭 IP가 나뉜 것 같다.

- 프라이빗 서브넷, 퍼블릭 서브넷 (aws)

![프라이빗 / 퍼블릭 서브넷 이미지](https://velog.velcdn.com/images/rockwellvinca/post/5bd96af0-15aa-47fd-a687-a662cc75140c/image.png)

  - `프라이빗 서브넷`

    - 프라이빗 서브넷은 인터넷 게이트웨이(Internet Gateway)를 통해 외부 인터넷과 직접 연결되지 않는 서브넷이다.

          외부에서 직접 접근할 수 없으며, 내부 통신이나 백엔드 서버, 데이터베이스 서버처럼
          보안이 중요한 리소스를 배치하는 데 사용된다.

          인터넷 접근이 필요한 경우에는 NAT 게이트웨이 또는 NAT 인스턴스를 통해 간접적으로 외부와 통신할 수 있다.

  - `퍼블릭 서브넷`

    - 퍼블릭 서브넷은 인터넷 게이트웨이를 통해 외부 인터넷과 직접 연결 가능한 서브넷이다.

          외부에서 접근이 필요한 웹 서버, Bastion Host(접속용 서버) 등을 배치하는 데 사용된다.

          퍼블릭 서브넷에 있는 인스턴스는 퍼블릭 IP를 가지고 있고, 인터넷 게이트웨이를 통해 직접 외부와 통신할 수 있다.

##### 퍼블릭 서브넷과 프라이빗 서브넷을 나누면,

##### 부 접근이 필요한 자원과 그렇지 않은 자원을 분리하여

##### 보안을 강화하고 네트워크 구성을 더 유연하고 효율적으로 만들 수 있다.

- `라우팅 개념 (간단하게)`

![라우팅 이미지](https://blog.kakaocdn.net/dn/n7G3v/btrmwPDKp4P/9yoS0rOohDnLCgfhPcv651/img.png)

  - `라우팅(Routing)`은 네트워크에서 데이터를 목적지까지 전달하기 위해 경로를 선택하는 과정이다.

        송신지에서 수신지까지 어떤 길로 데이터를 보낼지 결정하며,
        이 경로는 라우터(Router)가 가지고 있는 라우팅 테이블을 기반으로 선택된다.

        네트워크 간 통신을 가능하게 해주는 핵심 메커니즘이다.

(간단한 라우팅 과정)

  1. 송신 장치가 목적지 IP를 포함한 패킷을 생성하여 보낸다.

  2. 라우터는 목적지 IP를 보고 라우팅 테이블에서 가장 적합한 경로를 찾는다.

  3. 그 경로를 따라 패킷은 다음 라우터나 목적지로 전달된다.

      ##### 이 과정을 반복해 패킷은 최종 목적지에 도달한다.

##### 라우팅을 통해 복잡한 네트워크 안에서도
##### 데이터가 올바른 장치로 빠르게 전달될 수 있으며,
##### 네트워크 설계와 보안, 효율성에 있어 중요한 역할을 한다.

- `nat이란?`

![nat img](https://blog.kakaocdn.net/dn/dGQPWJ/btrfsdp49HD/TMwaHRijMiotBRT1DRJER0/img.png)

  - NAT(Network Address Translation)은 내부 네트워크의 프라이빗 IP를 퍼블릭 IP로 변환해주는 기술이다.

        내부 장치들이 인터넷에 접속할 수 있도록 해주며, 외부에서는 내부 IP를 직접 알 수 없도록 보호한다.

        주로 프라이빗 IP를 사용하는 내부 네트워크가 인터넷과 통신할 수 있도록 라우터나 NAT 게이트웨이에 설정된다.

(간단한 nat 동작 방식)

1. 내부 장치가 인터넷으로 요청을 보낼 때, NAT는 프라이빗 IP를 퍼블릭 IP로 바꿔서 외부로 보낸다.

2. 외부에서 응답이 오면 NAT는 그 응답을 원래의 내부 장치로 다시 전달한다.

이렇게 NAT는 프라이빗 IP과 퍼블릭 IP 변환을 통해
내부 장치가 인터넷을 사용할 수 있게 해주는 역할을 한다.

- `private ip를 갖고있는 pc가 외부 네트워크와 통신하는 과정`

  - (프라이빗 IP를 가지고 있는 PC의 인터넷 통신 과정을 간단하게 정리해보았습니다.)

        1. 사용자가 인터넷 접속 시도

        프라이빗 IP (ex. 192.168.0.10)를 가진 PC에서 구글 같은 외부 웹사이트에 접속 요청을 보냄.

        2. 패킷이 라우터/NAT 장비로 전달됨

        보낸 요청은 라우터 또는 NAT 장비로 전달됨.
        NAT 장비는 이 패킷의 출발지 (프라이빗) IP를 확인함.

        3. NAT가 IP 변환 수행

        NAT는 프라이빗 IP를 퍼블릭 IP(ex. 203.0.113.5)로 바꿔줌.

        `퍼블릭 IP로 바꾸면서 이 요청은 내부의 192.168.0.10이 보낸 것이라는 정보를 매핑 테이블에 저장함`

        4. 인터넷으로 요청 전송

        변환된 패킷이 외부 인터넷(구글 등)으로 전송되고고
        외부 서버는 퍼블릭 IP를 보고 응답함.

        5. 응답 도착 NAT가 다시 IP 변환

        외부 서버 응답 도착 시, NAT는 매핑 정보를 보고
        퍼블릭 IP을을 원래 프라이빗 IP로 다시 바꿔줌.

        6. PC로 응답 전달

        최종적으로 NAT가 응답을 내부 PC(ex. 192.168.0.10)에게 전달함.
        사용자는 웹사이트에 정상 접속됨.

정리하자면,

  - 프라이빗 IP는 인터넷에 직접 연결 안 된다.

  - NAT가 퍼블릭 IP로 바꿔서 통신하게 한다.

  - 외부 응답도 NAT가 다시 프라이빗 IP로 되돌려준다.

- `같은 대역에 있는 pc가 서로 통신하는 방법`

  - 같은 대역 : 서브넷 마스크로 계산했을 때 네트워크 주소가 동일한 IP끼리를 말한다.

    - 같은 대역 PC 간 통신 과정을 간단하게 정리해보았습니다.

          1. PC1이 PC2에게 데이터 전송 시도

          PC1은 PC2의 IP 주소(ex. 192.168.0.20)를 알고 있다.

          2. ARP 요청

          PC1은 PC2의 MAC 주소를 모르기 때문에, ARP(Address Resolution Protocol) 요청을 네트워크에 브로드캐스트로 보낸다.

          3. PC2가 ARP 응답

          PC2가 자신의 MAC 주소를 PC1에게 알려준다.

          이후 PC1은 이 정보를 ARP 캐시에 저장한다.

          4. 직접 통신 시작

          MAC 주소를 알게 된 PC1은 스위치를 통해 PC2에게 직접 이더넷 프레임을 전송한다.

          이때 전송하며 라우터를 거치지 않음, 스위치/허브 수준에서 통신된다.

          5. 양방향 통신 가능

          PC1 ↔ PC2 간 직접 통신이 가능하다.

          이후엔 ARP 캐시 덕분에 다시 ARP 요청 없이 바로 통신한다.

정리하자면,

  - 같은 대역이면 라우터 없이 직접 통신한다.

  - ARP를 이용해 상대방 MAC 주소를 알아낸 뒤 스위치를 통해 이더넷 프레임 단위로 직접 통신한다.

- `서로 다른 서브넷 대역에 있는 pc가 서로 통신하는 과정`

  - 다른 대역 : IP 주소와 서브넷 마스크를 비교했을 때 네트워크 주소가 서로 다른 경우를 다른 대역이라고 한다.

    - 다른 서브넷 대역에 있는 PC (PC1과 PC2의) 간 통신 과정을 간단하게 정리해보았습니다.

          1. PC1이 PC2에게 데이터 전송 시도

          PC1은 PC2의 IP 주소(192.168.2.20)를 알고 있다.

          2. 다른 서브넷임을 판단

          PC1은 자신의 서브넷 마스크와 비교해서
          PC2가 다른 네트워크 대역에 있다는 것을 인식한다.

          3. 기본 게이트웨이로 전송

          PC1은 라우터(기본 게이트웨이)의 MAC 주소로 프레임을 만들어 보낸다.

          목적지는 PC2의 IP지만, 실제 전달 대상은
          라우터이다.
          
          4. 라우터가 목적지 서브넷으로 전달

          라우터는 목적지 IP(192.168.2.20)를 보고
          자신의 라우팅 테이블을 참조하여 해당 서브넷으로 전달한다.

          5. PC2에게 전달 및 응답

          패킷이 목적지 서브넷에 도달하면,
          라우터는 PC2의 MAC 주소를 알아내어 전달한다. (필요 시 ARP 사용)

          6. PC2도 기본 게이트웨이를 통해 응답

          PC2도 PC1에게 응답할 때, 같은 방식으로 기본 게이트웨이 → PC1 순서로 전달한다.

정리하자면,

  - 다른 서브넷 간 통신은 라우터(기본 게이트웨이)를 반드시 거쳐야 함.

  - 직접 통신이 아닌 라우터를 통한 간접 통신이며, 라우팅 테이블과 ARP가 함께 작동해 최종 목적지까지 데이터가 전달된다.
